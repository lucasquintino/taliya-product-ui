## Batch 9 Non-Negotiables

- CRM patterns must be composed product components, not one-off screen markup.
- Components receive prepared props and callbacks only.
- No backend, API, auth, billing, routing, tenant, or agent runtime logic.
- Components must compose `@taliya/ui` primitives instead of duplicating primitive visuals.
- Missing layout/anatomy values must become `@taliya/tokens` before CRM CSS consumes them.
- Storybook must expose each CRM component as an inspectable story under `CRM / ...`.
- Image coverage stories remain later Batch 11 proof; Batch 9 stories prove component-level patterns.
