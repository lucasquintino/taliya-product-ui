## Extraction Rules

- Use the primary source for 1:1 component cloning first.
- Use secondary sources only to complete variants, states, or repeated usage.
- If the matrix and an image conflict on navigation labels, `navigation-contract.md` wins for labels and route families, while the image still wins for visual anatomy.
- If setup numbering conflicts, the exact approved image crop wins for each visual component. The broader setup block order remains a content-flow contract only and must not override a cloned component crop such as the Image 51A left stepper rail.
- Do not implement from this table alone. Open the actual image file from the canonical directory.
- Do not hardcode full screens. Extract reusable component anatomy and states.
